BIM Manager – Einrichtungsanleitung

Kollaborativer Betrieb mit Supabase · Geschätzte Einrichtungszeit: 10 Minuten

Wie funktioniert die Synchronisierung?
Alle Daten werden weiterhin lokal im Browser gespeichert (offline-fähig). Im Hintergrund synchronisiert bim_sync.js alle Änderungen automatisch in eure gemeinsame Supabase-Datenbank. Ein kleines Ampel-Icon in der Topbar jedes Moduls zeigt den Sync-Status.
1
Supabase-Konto anlegen

Geht auf supabase.com und erstellt ein kostenloses Konto. Der Free-Tier umfasst 500 MB Speicher und 2 GB Datentransfer pro Monat – mehr als ausreichend für dieses Tool.

→ supabase.com öffnen
Nur eine Person muss das Konto anlegen und die Zugangsdaten teilen. Kein weiterer Account nötig für den Kollegen.
2
Neues Projekt erstellen
  1. Nach dem Login auf „New project" klicken
  2. Projektname: z. B. bim-manager
  3. Datenbank-Passwort: sicheres Passwort notieren (wird später nicht mehr angezeigt)
  4. Region: Frankfurt (eu-central-1) wählen
  5. Plan: Free belassen → „Create new project"
Das Projekt-Setup dauert ca. 1–2 Minuten. Seite nicht schließen.
3
Datenbanktabelle anlegen

Im Supabase-Dashboard links auf „SQL Editor" klicken, dann diesen Code einfügen und ausführen:

CREATE TABLE bim_data (
  key       TEXT PRIMARY KEY,
  value     TEXT,
  updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- Zugriff für alle erlauben (Tool ohne Login)
ALTER TABLE bim_data ENABLE ROW LEVEL SECURITY;
CREATE POLICY "allow_all" ON bim_data
  FOR ALL USING (true) WITH CHECK (true);

Auf „Run" (▶) klicken. Ergebnis: Success. No rows returned

4
API-Zugangsdaten ermitteln

Links im Menü auf Settings → API klicken. Ihr benötigt:

  • Project URL – sieht aus wie: https://abcdefgh.supabase.co
  • anon public Key – langer String beginnend mit eyJh...
Der anon-Key ist für den öffentlichen Zugriff gedacht und kann im Source-Code stehen. Kein Problem für dieses interne Tool.
5
bim_sync.js konfigurieren

Öffnet die Datei bim_sync.js in einem Texteditor und ersetzt die zwei Zeilen oben:

// Vorher:
const BIM_SUPABASE_URL = 'DEINE_SUPABASE_URL';
const BIM_SUPABASE_KEY = 'DEIN_SUPABASE_ANON_KEY';

// Nachher (eure echten Werte):
const BIM_SUPABASE_URL = 'https://abcdefgh.supabase.co';
const BIM_SUPABASE_KEY = 'eyJhbGciOiJIUzI1NiIs...';
Wichtig: Speichert die Datei nach der Änderung. Dieselbe bim_sync.js teilt ihr dann mit eurem Kollegen – er muss sie nicht ändern.
6
Dateien teilen und öffnen

Schickt eurem Kollegen den kompletten Ordner (alle 8 Dateien). Die Struktur muss so aussehen:

📁 bim-manager/
  ├── bim_sync.js                   ← Sync-Adapter (mit euren Keys)
  ├── 01_projektverwaltung.html     ← Einstiegspunkt
  ├── 02_meilensteinplanung.html
  ├── 03_merkmalserver_v2.html
  ├── 04_anwendungsfaelle.html
  ├── 05_modellpruefung.html
  ├── 06_modellpruefung_regeln.html
  └── 07_berichtsgenerator.html

Beide öffnen 01_projektverwaltung.html im Browser. Beim ersten Start werden die Testdaten geladen und sofort in Supabase synchronisiert.

Sync-Status: In der Topbar jedes Moduls erscheint ein kleines Ampel-Icon:
● Grün = synchronisiert   ● Gelb = lädt   ● Rot = Fehler   ● Grau = lokaler Modus (kein Supabase)
7
Optional: Im Browser hosten (kein lokales Speichern nötig)

Falls ihr die Dateien nicht lokal speichern möchtet, könnt ihr das Tool kostenlos auf GitHub Pages hosten:

  1. GitHub-Konto erstellen (falls noch nicht vorhanden)
  2. Neues Repository anlegen: bim-manager
  3. Alle 8 Dateien hochladen
  4. Repository → Settings → Pages → Branch: main → Save
  5. URL: https://euer-name.github.io/bim-manager/01_projektverwaltung.html
Der Kollege muss dann nichts herunterladen – einfach den Link öffnen. Daten bleiben in Supabase synchronisiert.
